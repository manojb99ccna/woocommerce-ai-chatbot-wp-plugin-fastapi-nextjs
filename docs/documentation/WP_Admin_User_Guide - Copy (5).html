WooCommerce AI Customer Support

Client Documentation - WordPress Customer Side + React Admin User Side

Purpose

This guide explains how the system works from two perspectives only:

  • WP Side (Customer): Shopper opens chat and talks with AI.
  • Admin Side (React User): Store team monitors conversations and handles handoff.

1. WP Side (Customer Journey)

Step 1: Customer visits storefront page
The shopper opens any page where the chatbot widget is enabled in WordPress.
Step 2: Customer opens chat widget
User clicks the chat icon to open the support window.
Step 3: Customer submit name,email and open message textbox
Customer types name,email and submits it. and you see message box
Step 4: Customer sends message
Customer types a question (order, shipping, return, etc.) and submits it.

I type "Hi i want to buy product,please give me shopping page url."

Step 5: AI response is returned
The widget shows AI-generated answer in the conversation stream.
Step 7: Escalation request (if needed)
If customer asks for human support/refund/escalation, system marks it for admin follow-up.

If i type "I want to talk human."

Step 8: Escalation request button click then human support active.
If you click this button then request will send to admin email. also a google sheet also updated

2. WP Admin Side (Plugin Settings)

Step 1: Open WordPress Admin
Store admin logs in to WordPress and opens the chatbot plugin settings page.
Step 2: Enable or disable widget
Admin uses the plugin toggle option to enable or disable chatbot visibility on storefront pages.
Step 3: Enter backend API URL
Admin types the backend service endpoint URL in the provided field and saves settings.
Step 4: Verify storefront widget
After saving settings, admin opens storefront and confirms widget appears and sends messages correctly.

3. Admin Side (React User Journey)

Step 1: Admin opens React Login page.
User accesses the admin login page and signs in with credentials.

User goes to Conversations/Inbox to view all customer chats.
Step 4: Admin opens a conversation detail
User clicks one conversation to review full chat history between customer and AI.

If customer needs manual support or admin presses Start Handoff. First frontend support click admin email received email. in that link you can click after login you will see your escalation list.
Also if you direct login you also can see Escalation

If customer needs manual support, admin presses Start Handoff.
