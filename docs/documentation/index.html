WooCommerce AI Customer Support

Complete User Guide – WordPress Customer Side & React Admin Panel

Overview

This system provides automated customer support using AI, with an option for human takeover when needed.

  • Customer (Frontend): Interacts with AI chatbot.
  • Admin (Backend): Monitors conversations and handles escalations.

1. Customer Journey (WordPress Frontend)

Step 1: Visit Store Customer opens the website, and the chatbot widget appears on all pages if it is enabled from the backend.
Step 2: Open Chat Widget Click the chat icon to start.
Step 3: Enter Details Customer submits name and email to start chatting.
Step 4: Send Message Example:
Hi, I want to buy a product. Please share the shopping page URL.
Step 5: AI Responds AI generates an instant reply.
Step 6: Request Human Support Escalation is automatically triggered when the customer message matches certain conditions, such as:
  • Requesting a human (e.g., "I want to talk to a human", "agent", "representative")
  • Refund or cancellation requests (e.g., "refund", "return", "cancel my order")
  • Angry or abusive language (e.g., "worst", "scam", "unacceptable")
Example: I want to talk to a human.

When this happens, a "Request Human Support" button appears in the chat. When the customer clicks this button, an escalation request is sent to the admin via email, and the conversation is marked as escalated in the admin panel (Inbox). The admin can either open the conversation from the email link or directly log in to the admin panel to view all escalated conversations and start a response.

Step 7: Escalation Triggered Once the customer clicks the Request Human Support button, the system performs the following actions:
  • Sends an email notification to the admin team
  • Logs the escalation request using Make.com automation, which sends data to Google Sheets
  • Marks the conversation for human follow-up in the admin panel

2. WordPress Admin Settings

Step 1: Open Dashboard Login and open chatbot plugin settings.
Step 2: Enable/Disable Chatbot Toggle chatbot visibility.
Step 3: Set API URL Enter backend API endpoint and save.
Step 4: Verify Chatbot Confirm chatbot works on frontend.
