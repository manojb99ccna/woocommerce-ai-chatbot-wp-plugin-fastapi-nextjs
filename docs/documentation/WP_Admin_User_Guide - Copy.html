WooCommerce AI Customer Support

Client Documentation - WordPress Customer Side + React Admin User Side

Purpose

This guide explains how the system works from two perspectives only:

  • WP Side (Customer): Shopper opens chat and talks with AI.
  • Admin Side (React User): Store team monitors conversations and handles handoff.

1. WP Side (Customer Journey)

Step 1: Customer visits storefront page
The shopper opens any page where the chatbot widget is enabled in WordPress.
Image placeholder: Storefront page with floating chat icon visible at bottom-right.
Step 2: Customer opens chat widget
User clicks the chat icon to open the support window.
Image placeholder: Chat panel opened with greeting text and message input box.
Step 3: Customer sends message
Customer types a question (order, shipping, return, etc.) and submits it.
Image placeholder: Customer question shown in chat bubble.
Step 4: AI response is returned
The widget shows AI-generated answer in the conversation stream.
Image placeholder: AI response bubble under customer message.
Step 5: Escalation request (if needed)
If customer asks for human support/refund/escalation, system marks it for admin follow-up.
Image placeholder: Escalation state in chat (example: "A human agent will contact you").

2. Admin Side (React User Journey)

Step 1: Admin opens React dashboard
User accesses the admin login page and signs in with credentials.
Image placeholder: Admin login screen with username/password fields.
Step 2: Admin lands on dashboard
Main dashboard displays overall support activity and quick navigation.
Image placeholder: Dashboard cards/summary widgets.
Step 3: Admin opens conversation list
User goes to Conversations/Inbox to view all customer chats.
Image placeholder: Conversations table with IDs, status, and timestamps.
Step 4: Admin opens a conversation detail
User clicks one conversation to review full chat history between customer and AI.
Image placeholder: Conversation detail page with message thread.
Step 5: Admin starts handoff (human takeover)
If customer needs manual support, admin presses Start Handoff.
Image placeholder: Start Handoff button highlighted in conversation page.
Step 6: Admin sends manual reply
Admin types reply and sends direct response to customer conversation.
Image placeholder: Admin reply box and sent message in thread.
Step 7: Admin ends handoff
After issue is solved, admin clicks End Handoff to return control to AI flow.
Image placeholder: End Handoff action with updated conversation status.

3. Quick UAT Checklist

  • Customer can open widget on storefront.
  • Customer message is sent and AI reply is visible.
  • Escalation scenario can be triggered from customer chat.
  • Admin can login and view conversation list.
  • Admin can open conversation and perform Start/End Handoff.
  • Admin manual reply appears in thread correctly.
